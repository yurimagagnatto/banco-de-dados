Vamos criar um estudo de caso fictício para um sistema de gestão de projetos e tarefas para a criação de um banco de dados relacional.

## Estudo de Caso: Sistema de Gestão de Projetos e Tarefas

Uma empresa de desenvolvimento de software, "Tech Solutions", deseja implementar um sistema de gestão de projetos e tarefas para otimizar o fluxo de trabalho de sua equipe de desenvolvimento. O sistema deve permitir o acompanhamento de projetos, atribuição de tarefas aos membros da equipe, monitoramento do progresso e geração de relatórios.

## Requisitos Funcionais

### Cadastro de Projetos

O sistema deve permitir o cadastro de novos projetos, incluindo informações como nome do projeto, descrição, data de início e data de término prevista.

### Gestão de Tarefas

Cada projeto pode ter várias tarefas associadas. As tarefas devem ter um nome, descrição, status (pendente, em andamento, concluída), data de início e data de conclusão prevista.

### Atribuição de Tarefas

Os membros da equipe devem ser capazes de ser atribuídos a tarefas específicas em um projeto.

### Monitoramento de Progresso

O sistema deve permitir o acompanhamento do progresso de cada tarefa e do projeto como um todo.

### Geração de Relatórios

Deve ser possível gerar relatórios sobre o progresso do projeto, tarefas concluídas, tarefas pendentes, etc.

## Atividade

### 1. Modelo Conceitual (Criação do Modelo Entidade-Relacionamento)

Desenhe um modelo entidade-relacionamento que represente as entidades principais e seus relacionamentos no sistema de gestão de projetos e tarefas.

### 2. Modelo Lógico (Criação do Modelo Entidade-Relacionamento)

Transforme o modelo entidade-relacionamento em um modelo lógico, especificando as tabelas, atributos e relacionamentos, incluindo chaves primárias e estrangeiras.
